# Dashboard (React + Vite)

This app visualizes enriched Luma event data and LinkedIn joins generated by the data build pipeline.

## What data it expects

The app reads a generated JSON file at `dashboard/src/data/dashboard.json`.

That file is built from:
- `luma-enricher enrich` output JSON
- LinkedIn `Connections.csv`
- LinkedIn `messages.csv`
- optional `custom-data/connection_locations.csv`

## Prerequisites

- Bun 1.0+ (`bun --version`)
- Node 20+ (for the parent `luma-enricher` project)
- A generated enriched file from the CLI (example: `/tmp/luma-enriched.json`)

## Build local dashboard data

From `luma-enricher/dashboard/`:

```bash
bun install
bun run build:data \
  --enriched /tmp/luma-enriched.json \
  --connections /absolute/path/to/linkedin-data-export/Connections.csv \
  --messages /absolute/path/to/linkedin-data-export/messages.csv \
  --locations /absolute/path/to/custom-data/connection_locations.csv \
  --out ./src/data/dashboard.json
```

Notes:
- `--locations` is optional. If missing, the script continues without location enrichment.
- Run `bun run build:data:help` for all options and env var fallbacks.
- If you omit flags, the script tries defaults relative to this repo layout.

## Run the dashboard

From `luma-enricher/dashboard/`:

```bash
bun run dev
```

Then open the local Vite URL (usually `http://localhost:5173`).

## One-command dev flow

If your file paths match repo defaults, you can run:

```bash
bun run dev:with-data
```

This rebuilds `src/data/dashboard.json` and starts Vite.

## Troubleshooting

- `Enriched JSON not found`: run the CLI enrich step first.
- `LinkedIn Connections CSV not found`: pass explicit `--connections`.
- Dashboard starts but looks empty: confirm `dashboard/src/data/dashboard.json` exists and has non-zero `stats.total_events`.
